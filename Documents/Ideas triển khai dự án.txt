Mô hình tổng quan: Client sever (Liên kết các máy) (Socket)

I. Client
	- Chat 
		+ Chat giữa máy khách và máy qly
	- Order (Poly coffee) 
		+ Khi máy khách order xong sẽ thông báo về máy quản lý
	- Đăng xuất
		+ Khi đăng xuất thì tự động tính giờ và tính tiền máy
	- Đổi mật khẩu 
		+ Cho phép người dùng đổi mật khẩu của họ
	- Quên mật khẩu
		+ Cho phép đặt lại mkhau

II. Admin
1. Quản lý máy:
	- Sơ đồ máy 
		+ Máy trống = màu xám - Máy hoạt động = Màu xanh
	- ID của khách
	- Giờ bắt đầu
	- Giờ kết thúc

2. Quản lý game:
	- Icon (Ảnh của game đó)
	- Tên game
	- Tình trạng: Hoạt động/Ngừng hoạt động
	- Ngày kích hoạt (Ngày cài đặt game đó)
	- CRUD game từ admin (Thêm/Ẩn game)

3. Quản lý sản phẩm
	*Áp dụng PolyCafe
	- Icon
	- Mã SP
	- Tên SP
	- Loại
	- Giá
	- Trạng thái (Đang bán/Ngừng bán)
	- Giao diện thêm/sửa/ẩn món

4. Hóa đơn
	*Khi máy out id
	- Tổng giờ chơi (Tự tính) => Trừ tiền trong id khách

	*Áp dụng khi khách order
	- Tổng tiền đồ uống => Nhân viên
	- Thanh toán => In hóa đơn

5. Thống kê
	- Doanh thu (Tổng tiền máy/Tổng tiền sản phẩm)
	- Sản phẩm bán chạy (Đồ ăn/Đồ uống)
	- Máy hoạt động nhiều nhất
	- Top account hoạt động nhiều nhất

6. Nhận thông báo từ máy khách
	*Nhận thông báo về các sự kiện như:
	- Order
	- Chat
	- Hoạt động trên máy (Đang chơi game gì)
	- Đăng xuất
